# Why this project

Static publishing is often treated as a simple frontend delivery problem. For nonprofit, educational, archive, and public-interest projects, the harder problem is governance: how to preserve stable identity, content integrity, public-safe output, source references, and release accountability without operating a heavy database-backed platform.

Static CID Governance addresses this gap by combining:

- stable logical identity;
- content-derived CID snapshots;
- source-reference metadata;
- public-safe projection;
- release-gate verification;
- package inventory guarding;
- Offline Editor public reference components.

This project is FHIR-informed, but it is not a FHIR implementation, FHIR profile, FHIR Implementation Guide, or non-medical edition of FHIR. It selectively adapts concepts such as logical identity, versioned reference, human-readable narrative, controlled coding, and provenance-style auditability into a static publishing context.

The goal is to provide an open reference pattern that nonprofit organizations and public-interest teams can study, adopt, audit, improve, and adapt.
